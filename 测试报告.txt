核心赚钱逻辑:
  a) 每笔交易都有止损保护,亏损可控(最大单笔亏损 = 本金 × 3%)
  b) 止盈大于止损(S1:2.0/1.8=1.11倍, S5:1.0/0.8=1.25倍)
     只要胜率超过临界值(约45%)即可实现正期望收益
  c) 六套策略覆盖不同行情,通过自动切换避免在不适的市场中亏钱
  d) 每笔下单前仓位由风控自动计算: qty = 本金×风险% / (止损距离%×价格)
     确保单笔亏损永远不超过本金3%


2. 爆仓的极限是多少？风控有哪些？
--------------------------------------------------------------------------------
【爆仓极限】
爆仓取决于杠杆、仓位和止损距离三个因素。系统通过以下机制将爆仓概率降到极低:

  · 默认最大杠杆: 10倍(可在设置中调整)
  · 单笔最大亏损: 本金的 3%(默认,可在设置中调整)
  · 保证金上限: 不超过本金的 80%
  · 自动仓位计算: 根据止损距离和风险上限反推下单数量

  举例: 本金$10,000, 杠杆10x, 止损1.8%
  单笔最大亏损 = $10,000 × 3% = $300
  仓位 = $300 ÷ (1.8% × 价格 × 10x) → 自动计算确保单笔亏损≤$300
  要爆仓需要连续亏损 333 笔(假设每笔亏满3%),几乎不可能

【风控网关(三道防线)】

  第一道: 单笔亏损限制
    下单前估算预期亏损 = 止损距离% × 名义价值 × 杠杆
    若 > 本金 × 3% → 拒绝下单

  第二道: 当日累计亏损限制
    当日已实现亏损累计 > 本金 × 15% → 自动停盘
    当天剩余时间无法再开新仓,次日 00:00 自动重置

  第三道: 保证金上限
    保证金需求 > 本金的 80% → 拒绝下单
    防止过度使用杠杆,留足缓冲

【前端可自定义的风控参数】
  初始本金 / 最大杠杆 / 单笔最大亏损% / 当日最大亏损% / 最小止损距离%
  均可在「设置」页调整,修改后即刻生效。


3. 每个模块的作用是什么？怎么运行的？
--------------------------------------------------------------------------------

【数据引擎】cryptoagents/data/
  · ccxt_fetcher.py: 通过 CCXT 统一接口连接交易所(Binance/OKX/Bybit)
    拉取 OHLCV K线数据,支持分页(突破单次1000根限制)
    回测/图表用生产公开数据(无需密钥)
  · kline_converter.py: 将K线转为系统内部 F/S/L/H 紧凑格式
    F=实时价 S=前收盘 L=最低 H=最高
    给 AI 分析用,压缩数据量同时保留关键信息
  · indicator_calc.py: 计算技术指标
    MACD(12,26,9) / RSI(7) / BOLL(20,2σ) / KDJ(9)

【策略引擎】cryptoagents/strategy/
  · strategies.py: 6套内置策略 + CustomStrategy(参数化自定义策略)
    每根K线调用 calculate() 返回 BUY/SELL/HOLD 信号
  · scheduler.py: 策略调度器(状态机)
    AI感知到行情变化 → 查市场状态映射表 → 推荐策略
    → 若与当前不同 → 前端弹窗确认(或自动切换)

【回测引擎】cryptoagents/backtest/
  · engine.py: 逐根K线模拟交易
    流程: 拉历史数据 → 逐根计算信号 → 模拟入场/检查止损止盈/平仓
    含: 手续费(0.04%) / 滑点(0.01%) / 杠杆放大 / 净持仓模型
    结果: 总收益 / 胜率 / 盈亏比 / 最大回撤 / 权益曲线 / 交易明细
    普速: 1x~100x(只控制进度刷新频率,信号计算每根都执行)

【AI引擎】cryptoagents/ai/
  · trend_analyzer.py: DeepSeek API调用
    输入: 近20根15m + 10根1h K线(F/S/L/H) + MACD/RSI/BOLL/KDJ
    输出: 市场状态(8种) + 置信度 + 推荐策略 + 建议杠杆
  · ai_service.py: 趋势感知编排层
    拉数据 → AI分析 → 存报告(SQLite) → 通知调度器 → 可选自动交易
  · chat_service.py: AI对话助手
    注入实时K线+新闻上下文,支持自然语言创建策略
  · news_service.py: 金融新闻抓取
    从 CoinDesk/CoinTelegraph RSS 抓取(免费,2分钟缓存)

【风控引擎】cryptoagents/risk/
  · gateway.py: 三道防线(单笔/当日/保证金),下单前强制检查

【执行引擎】cryptoagents/execution/
  · executor.py: 三模式交易
    paper: 本地模拟撮合(paper_account),不接触交易所
    testnet: 交易所沙箱,用模拟资金真实下单
    live: 真实资金,风险自负
  · paper_account.py: 模拟盘本地账户
    持仓/盈亏记录到 SQLite,支持逐仓平仓/一键全平

【调度引擎】app/worker/
  · scheduler_worker.py: APScheduler 后台任务
    每15分钟(可自定义)对所有监控币种执行 AI趋势感知
    前台「AI报告」可查看历史,「AI助手」页可控制开关和间隔

【Web API】app/routers/
  · 10个路由模块: kline/strategy/backtest/trend/trade/risk/
    scheduler/config/ai/settings
  · 全部端点前缀 /api/v1

【前端】frontend/
  · Vue 3 + TailwindCSS 单页应用(5个Tab):
    主面板 / AI助手 / 新闻 / AI报告 / 策略 / 回测 / 设置
  · 生产构建后由 FastAPI 直接服务,无需 Node 服务器

  ─────────────────────────────────────────────
  STRONG_BULL       S1         S5         仅做多

  TREND_CHANGE      S3         S1         双向

【自动切换流程】(全自动化链路)

  第1步: APScheduler 定时器每15分钟触发
  第2步: ai_service.run_sensing_all()
         对监控列表所有币种: 拉15m+1h K线 → 计算技术指标
         → 拼装 Prompt → 调用 DeepSeek API
  第3步: AI 返回JSON: market_state + confidence + recommended_strategy
  第4步: scheduler.on_trend_change()
         查上表,若推荐策略 ≠ 当前策略
         → 默认: 前端弹窗「建议切换策略」,等待用户确认/拒绝
         → 自动模式(前端开关): 直接切换并可选自动下单

  注:策略切换需人工确认是默认安全设置,可在「设置」页关闭。


5. 态势感知的置信度是什么？什么数值适合什么交易？
--------------------------------------------------------------------------------
【置信度(confidence)的含义】
置信度是 DeepSeek AI 对自己判断可信程度的自我评估,范围 0%~100%:

  · 100% = AI认为判断非常确定
  · 50% = AI认为判断有较大不确定性
  · 0% = 完全不确定

AI根据K线的清晰度、指标一致性、是否有明确的形态等因素给出。

【置信度→交易建议】

  置信度     建议操作
  ──────────────────────────────────────
  80%~100%   高确定性行情,可正常下单
            信号清晰,趋势/形态明确
  60%~80%   中等确定性,可下单但建议降低杠杆
            例如:建议杠杆 = 默认杠杆 × 置信度
  40%~60%   低确定性,仅做参考不交易
            AI返回的策略切换建议可能不够可靠
  0%~40%   不确定性高,不建议任何操作
            可能是无趋势震荡/数据不足

实际使用中:
  · 趋势行情(STRONG_BULL/BEAR)普遍置信度 65%-90%
  · 震荡行情(RANGING)普遍置信度 45%-65%
  · 极端行情(CRASH_BOUNCE/PUMP_REVERSAL)置信度 55%-75%

【AI输出字段完整说明】
  market_state: 8种市场状态之一
  confidence: 置信度
  recommended_strategy: 推荐策略编号(S1~S6)
  reasoning: AI的分析推理文本
  risk_level: LOW / MEDIUM / HIGH
  suggested_leverage: 建议杠杆倍数
  key_levels: {"support": 支撑价, "resistance": 阻力价}

7. 整个软件运行的逻辑是什么？
--------------------------------------------------------------------------------
【启动流程】
  1. main.py lifespan 启动
  2. init_db() → 创建 SQLite 数据库(首次则建表)
  3. settings_store.apply_to_settings() → 从 SQLite 加载用户保存的配置
  4. start_scheduler() → APScheduler 启动15分钟循环定时任务
  5. (exe模式) 自动打开浏览器 → http://127.0.0.1:8000
  6. 前端 SPA 加载,5个Tab就绪

【运行时数据流】

  ┌────────────────────────────────────────────────┐
  │                  定时器 (每15分钟)                │
  └────────────────────┬───────────────────────────┘
                       ↓
  ┌─────────────────────────────────────────────────┐
  │ ai_service.run_sensing(symbol)                   │
  │  1) CCXTFetcher 拉取 Binance/OKX/Bybit K线       │
  │  2) indicator_calc 计算 MACD/RSI/BOLL/KDJ        │
  │  3) kline_converter 转为 F/S/L/H 格式            │
  │  4) 调用 DeepSeek API 分析                       │
  │  5) 解析 JSON 结果 → TrendResult                 │
  │  6) 持久化到 SQLite (ai_reports表)               │
  └────────────────────┬────────────────────────────┘
                       ↓
  ┌─────────────────────────────────────────────────┐
  │ StrategyScheduler.on_trend_change()              │
  │  查 STRATEGY_MARKET_MAP 表                       │
  │  → AI推荐的策略 ≠ 当前策略?                       │
  │    ├─ 是→前端弹窗「建议切换策略」                   │
  │    │    ├─ 用户确认: 切换策略 → 如有自动交易开关   │
  │    │    │             → 执行策略信号下单           │
  │    │    └─ 用户拒绝: 保持当前                      │
  │    └─ 否→保持不变                                  │
  └─────────────────────────────────────────────────┘

【用户手动操作流程】

  策略页:
    选择币种 → 计算所有策略信号 → 查看BUY/SELL/HOLD
    → 点「一键下单」→ 执行下单流程
    → 或点「激活」切换当前活跃策略

  下单流程:
    strategy_order(post) → 拉K线 → 策略calculate() → 
    → 信号为BUY/SELL → 风控网关检查 →
    → 通过 → 仓位计算 → 
    → paper: paper_account模拟撮合
    → testnet/live: CCXT真实下单

  回测流程:
    设置参数(币种/策略/周期/起始结束/杠杆/倍速)
    → 提交 → 后台线程逐根模拟 →
    → 前端轮询进度(每0.5秒) → 进度条+当前权益 →
    → 完成 → 显示总收益/胜率/盈亏比/最大回撤/交易明细

  AI助手:
    输入问题 → AI拿到实时K线+新闻 → 分析回复
    → 若描述策略需求 → AI生成JSON策略规格 →
    → 用户点「创建此策略」→ 存入数据库 + 立即可用

  新闻:
    每2分钟从 CoinDesk/CoinTelegraph RSS 自动刷新
    → 前端列表展示 → AI聊天时可注入新闻上下文

【数据存储】
  所有数据存储于 SQLite (data/cryptoagents.db):
    · ai_reports: AI趋势报告历史
    · ai_trend_log: 趋势分析记录
    · trade_log: 交易记录
    · backtest_results: 回测结果
    · paper_orders: 模拟盘持仓/订单
    · app_settings: 用户自定义配置(15项)
    · custom_strategies: 用户创建的自定义策略

【配置管理】
  15项可配参数持久化到 SQLite + .env(API密钥):
    · 交易模式(paper/testnet/live) → 运行时即时生效
    · 风控参数 → 运行时即时生效
    · AI参数(模型/温度/间隔) → 运行时即时生效
    · 监控币种列表 → 运行时即时生效
    · API密钥 → 写入.env持久化,重启不丢失
