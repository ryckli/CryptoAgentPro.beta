║  一、现状审计                                                                 ║
╚══════════════════════════════════════════════════════════════════════════════╝

CCXT渗透了12个文件、约63处调用，分为两大类：

  类别A — 公开行情数据 (无需密钥, 7个文件):
    · fetch_ohlcv()     拉K线     (回测/图表/AI感知/策略信号/组合)
    · fetch_ticker()    拉现价     (下单取价/持仓估值/止损止盈)
    · load_markets()    加载交易对  (连通性检测)

  类别B — 私有交易接口 (需密钥, 2个文件):
    · create_market_order()  市价开仓/平仓  (executor.py)
    · fetch_positions()      查持仓        (executor/autopilot/trade)
    · set_leverage()         设杠杆        (executor.py)
    · set_sandbox_mode()     切测试网      (executor/ccxt_fetcher)

  涉及文件清单:
    1. cryptoagents/data/ccxt_fetcher.py        ← 核心数据层 (将被替换)
    2. cryptoagents/execution/executor.py       ← 下单引擎 (将被替换)
    3. cryptoagents/backtest/engine.py          ← 回测 (只调fetch_ohlcv_range)
    4. cryptoagents/execution/portfolio.py      ← 组合 (只调fetch_ohlcv)
    5. cryptoagents/execution/paper_account.py  ← 模拟盘 (无CCXT, 不动)
    6. app/routers/kline.py                     ← K线API (调CCXTFetcher)
    7. app/routers/strategy.py                  ← 策略信号 (调CCXTFetcher)
    8. app/routers/trade.py                     ← 交易API (调CCXTFetcher取价)
    9. app/routers/autopilot.py                 ← 自动驾驶 (调CCXTFetcher)
   10. cryptoagents/ai/ai_service.py            ← AI感知 (调CCXTFetcher)
   11. app/main.py                              ← WebSocket (调CCXTFetcher)
   12. cryptoagents/backtest/monte_carlo.py     ← 蒙特卡洛 (间接通过engine)

  │ ├── __init__.py                                             │

  设计原则:
    · 统一接口: 所有交易所实现同一个 ExchangeBase 抽象类
    · 零依赖: 仅用 httpx (已在requirements中) + hmac/hashlib (标准库)
    · 向下兼容: 保留 CCXTFetcher 类名作为别名, 旧代码零改动
    · 自动故障转移: OKX不通自动切Binance


╔══════════════════════════════════════════════════════════════════════════════╗
║  三、统一接口定义 (ExchangeBase)                                              ║
╚══════════════════════════════════════════════════════════════════════════════╝

  class ExchangeBase(ABC):
      # --- 公开行情 (无需密钥) ---
      def fetch_ohlcv(symbol, timeframe, limit, since) -> pd.DataFrame
      def fetch_ticker(symbol) -> dict  # {"last": float, "bid": float, "ask": float, "timestamp": int}
      def fetch_tickers(symbols) -> dict  # 批量现价
      def load_markets() -> dict  # 交易对信息

      # --- 私有交易 (需密钥) ---
      def fetch_positions(symbols) -> list[dict]  # 持仓列表
      def place_market_order(symbol, side, qty, params) -> dict  # {"id": str, "status": str}
      def close_position(symbol, side, qty) -> dict  # reduceOnly平仓
      def set_leverage(symbol, leverage) -> dict
      def fetch_account_balance() -> dict  # {"total": float, "available": float}

      # --- 辅助 ---
      def milliseconds() -> int
      @property
      def rate_limit_ms() -> int


╔══════════════════════════════════════════════════════════════════════════════╗
║  四、OKX V5 API 实现 (okx.py)                                                ║
╚══════════════════════════════════════════════════════════════════════════════╝

  文档: https://www.okx.com/docs-v5/zh/

  Base URL:
    生产: https://www.okx.com
    模拟: https://www.okx.com (header: x-simulated-trading: 1)

  认证 (HMAC SHA256):
    timestamp = ISO8601格式
    sign = base64(hmac_sha256(secret, timestamp + method + requestPath + body))
    Headers:
      OK-ACCESS-KEY: api_key
      OK-ACCESS-SIGN: sign
      OK-ACCESS-TIMESTAMP: timestamp
      OK-ACCESS-PASSPHRASE: passphrase  (即EXCHANGE_PASSWORD)
      x-simulated-trading: 1  (仅模拟盘)

  接口映射:
  ┌──────────────────┬──────────────────────────────────────────────┐
  │ 功能             │ OKX端点                                       │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ K线              │ GET /api/v5/market/candles?instId=&bar=&limit=│
  │                  │ bar映射: 15m→15m, 1h→1H, 4h→4H, 1d→1D       │
  │ 现价             │ GET /api/v5/market/ticker?instId=             │
  │ 持仓             │ GET /api/v5/account/positions?instId=         │
  │ 下单             │ POST /api/v5/trade/order                      │
  │                  │ body: {instId, tdMode:"cross", side, ordType: │
  │                  │  "market", sz, reduceOnly, attachAlgoOrds:    │
  │                  │  [止盈止损]}                                  │
  │ 设杠杆           │ POST /api/v5/account/set-leverage             │
  │                  │ body: {instId, lever, mgnMode:"cross",       │
  │                  │  posSide:"long"/"short"}                     │
  │ 账户余额         │ GET /api/v5/account/balance                   │
  │ 交易对           │ GET /api/v5/public/instruments?instType=SWAP  │
  └──────────────────┴──────────────────────────────────────────────┘

  OKX 特殊处理:
    · 合约单位: OKX的sz是"张数", 需要通过ctVal(每张面值)换算
    · 止盈止损: OKX用attachAlgoOrds附加止盈止损算法订单
    · 持仓方向: OKX返回posSide="long"/"short"
    · symbol格式: OKX用 BTC-USDT-SWAP, 需从 BTC/USDT 转换


╔══════════════════════════════════════════════════════════════════════════════╗
║  五、Binance Futures API 实现 (binance.py)                                   ║
╚════════════─────────────────────────────────────────────────────────────────╝

  文档: https://developers.binance.com/docs/binance-spot-api-docs

  Base URL:
    生产: https://fapi.binance.com
    测试: https://testnet.binancefuture.com

  认证 (HMAC SHA256):
    timestamp = 毫秒时间戳
    sign = hmac_sha256(secret, queryString)
    Header: X-MBX-APIKEY: api_key

  接口映射:
  ┌──────────────────┬──────────────────────────────────────────────┐
  │ 功能             │ Binance端点                                   │
  ├──────────────────┼──────────────────────────────────────────────┤
  │ K线              │ GET /fapi/v1/klines?symbol=&interval=&limit=  │
  │                  │ interval: 15m, 1h, 4h, 1d                    │
  │ 现价             │ GET /fapi/v1/ticker/price?symbol=             │
  │ 持仓             │ GET /fapi/v2/positionRisk?symbol=             │
  │ 下单             │ POST /fapi/v1/order                           │
  │                  │ params: {symbol, side, type:"MARKET",         │
  │                  │  quantity, reduceOnly, stopPrice,             │
  │                  │  closePosition}                              │
  │ 设杠杆           │ POST /fapi/v1/leverage?symbol=&leverage=      │
  │ 账户余额         │ GET /fapi/v2/balance                          │
  │ 交易对           │ GET /fapi/v1/exchangeInfo                     │
  └──────────────────┴──────────────────────────────────────────────┘

  Binance 特殊处理:
    · 止盈止损: Binance用单独的STOP/TAKE_PROFIT订单类型
    · 持仓: positionRisk返回positionAmt(正=多/负=空)
    · symbol格式: BTCUSDT (无斜杠)


╔══════════════════════════════════════════════════════════════════════════════╗
║  六、迁移策略 (零破坏性)                                                      ║
╚════════─────────────────────────────────────────────────────────────────────╝

  策略: 保留 CCXTFetcher 类名, 内部改为调用新的原生模块。

  ┌────────────────────────────────────────────────────┐
  │ cryptoagents/data/ccxt_fetcher.py (兼容层)          │
  │                                                     │
  │  旧代码: import ccxt → fetch_ohlcv → fetch_ticker  │
  │  新代码: from cryptoagents.exchange import factory  │
  │         fetch_ohlcv → factory.get().fetch_ohlcv    │
  │         fetch_ticker → factory.get().fetch_ticker  │
  │                                                     │
  │  对外接口完全不变, 12个文件零改动                    │
  └────────────────────────────────────────────────────┘

  同时修改 executor.py 的 _exchange() 方法:
    旧: ex = getattr(ccxt, name)(opts); ex.create_market_order(...)
    新: ex = factory.get_exchange(name, keys); ex.place_market_order(...)

  executor.py 是唯一需要改业务逻辑的文件 (CCXT方法名 → 新方法名)。

╔══════════════════════════════════════════════════════════════════════════════╗
║  八、风险与收益                                                               ║
╚══════════──────────────────────────────────────────────────────────────────╝

  收益:
    · 去掉ccxt依赖 (省30MB+, exe从111MB降到~80MB)
    · 完全掌控API调用逻辑, 不受ccxt版本升级影响
    · OKX止盈止损用原生attachAlgoOrds, 比ccxt的params更可靠
    · 可针对交易所特性做优化 (如OKX的WebSocket推送)

  风险:
    · OKX/Binance API变更需手动维护 (但频率很低)
    · 合约张数换算需精确 (OKX的ctVal)
    · 签名算法必须100%正确 (否则认证失败)

  回退方案:
    保留ccxt_fetcher.py的旧代码在注释中, 如新模块出问题可一键回退
