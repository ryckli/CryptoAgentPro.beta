╔══════════════════════════════════════════════════════════════════════════════╗
║  M1: AI自适应参数优化器                                                      ║
╚══════════════════════════════════════════════════════════════════════════════╝

设计目标: 策略不再是固定参数的死物, 而是会根据市场环境和近期表现自动进化的
          活体。每30秒检查一次每个策略的短期表现, 若出现连续亏损或信号质量
          下降, AI介入分析原因并提出参数调整建议。

━ 核心架构 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  cryptoagents/ai/adaptive_optimizer.py    ← 新增

  类: AdaptiveOptimizer
  ┌─────────────────────────────────────────────┐
  │ 每30秒循环 (独立后台线程)                    │
  │                                              │
  │ for each strategy in watchlist:              │
  │   1. 拉取最近N笔交易 (从paper_orders)         │
  │   2. 计算滚动指标 (过去2H窗口)                │
  │      · Sharpe Ratio = (μ/σ) × √252          │
  │      · Win Rate rolling(20 trades)           │
  │      · Avg Profit/Loss ratio                 │
  │      · 连续亏损次数                          │
  │   3. 判断是否需要优化:                        │
  │      · 连续亏损≥3笔 → 紧急优化               │
  │      · Sharpe < 0 → 常规优化                 │
  │      · WinRate 下降>30% → 信号质量优化        │
  │   4. 调用 DeepSeek 分析并建议新参数           │
  │      → 喂入: 当前参数 + 近期表现 + 市场状态   │
  │      → 输出: 新参数建议 + 调整理由            │
  │   5. 热更新策略参数 (动态替换策略实例属性)     │
  │   6. 记录参数变更历史 (adapt_log表)           │
  └─────────────────────────────────────────────┘

  Prompt设计:
  """
  你是量化策略参数优化专家。当前策略 {id}: {name}, 参数: {params}
  过去2小时表现: 胜率{wr}%, Sharpe{sharpe}, 连续亏损{streak}笔
  当前市场: {market_state}, 置信度{conf}%
  请分析是否需要调整参数,输出JSON:
  {"need_adapt":true/false, "new_params":{...}, "reason":"..."}
  可调参数范围:
    S1: fast[3-20], slow[15-50], sl_pct[0.5-3], tp_pct[0.5-5], sep[0.05-0.5]
    S2: period[5-21], oversold[10-40], overbought[60-90], sl_pct[0.5-3], tp_pct[0.5-5]
    S3: vol_mult[1.0-3.0], sl_pct[0.5-3], tp_pct[0.5-5]
    S4: rsi_thresh[10-35], drop_pct[1.0-8.0], tp_pct[0.5-5]
    S5: ema_period[10-50], sl_pct[0.3-2], tp_pct[0.5-3]
    S6: td_count[7-13], sl_pct[0.5-3], tp_pct[0.5-5]
  调整原则: 高波动环境偏宽止损, 低波动环境偏窄止损; 趋势行情加大止盈,
            震荡行情减小止盈; 连续亏损时先缩小仓位(实际通过增大止损距离实现)
  """

  安全约束:
    · 止损只能增大不能减小(保护性约束, 防止AI在连续亏损后更激进)
    · 止盈可以在区间内自由调整
    · 每次调整幅度≤原值的30%(防止单次过度调整)
    · 两次调整之间至少间隔5分钟(防止震荡调整)

  数据存储 (SQLite新表):
    adapt_log: id, ts, strategy_id, old_params, new_params, reason,
               performance_snapshot (JSON), applied (bool)


╔══════════════════════════════════════════════════════════════════════════════╗
║  M2: 多币种组合优化器                                                        ║
╚══════════════════════════════════════════════════════════════════════════════╝

设计目标: 从单一币种独立交易升级为多币种组合管理。基于现代组合理论(MPT)和
          Kelly准则, 在不同币种之间动态分配资金, 最大化风险调整后收益。

━ 核心架构 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  cryptoagents/execution/portfolio.py    ← 新增

  类: PortfolioManager
  ┌─────────────────────────────────────────────┐
  │ 核心方法:                                    │
  │                                              │
  │ 1. calculate_returns(symbols, lookback_1h)   │
  │    从CCXT拉各币种1H收益序列                   │
  │    → 返回 {symbol: [ret1, ret2, ...]}        │
  │                                              │
  │ 2. correlation_matrix(returns)               │
  │    Pearson相关系数矩阵                        │
  │    → 用于判断分散化程度                       │
  │                                              │
  │ 3. allocate_risk_budget(capital, symbols)     │
  │    风险平价(Equal Risk Contribution):         │
  │    每个币种对组合的边际风险贡献相等            │
  │    → 波动率高的币种分配更少资金               │
  │    → 波动率低的币种分配更多资金               │
  │    → 相关性高的币种视为一个风险块             │
  │    → 返回 {symbol: weight}                   │
  │                                              │
  │ 4. kelly_bet_sizing(win_rate, avg_win,       │
  │                     avg_loss, capital)        │
  │    Kelly分数: f* = (p*b - q) / b             │
  │    p=胜率, q=1-p, b=平均盈利/平均亏损        │
  │    实际使用1/2 Kelly (半凯利, 更保守)          │
  │    → 返回 每个币种的建议开仓金额              │
  │                                              │
  │ 5. rebalance(positions, target_weights)       │
  │    计算当前头寸与目标权重偏差                  │
  │    → 超配减仓, 低配加仓                       │
  │    → 返回调整方案列表                         │
  └─────────────────────────────────────────────┘

  资金分配层次:
  ┌─────────────────────────────────────────┐
  │ 总资金 $10,000                           │
  ├─────────────────────────────────────────┤
  │ Layer 1: 风险平价分配                    │
  │   BTC($4,000) ETH($3,500) SOL($2,500)   │
  ├─────────────────────────────────────────┤
  │ Layer 2: Kelly下注 (单个仓位)            │
  │   Kelly分数 × 分配金额 = 单笔开仓金额    │
  ├─────────────────────────────────────────┤
  │ Layer 3: 风控约束 (不可绕过的天花板)      │
  │   单笔亏损 ≤ 分配金额 × 3%               │
  │   总敞口 ≤ 本金 × 80%                    │
  └─────────────────────────────────────────┘

  再平衡触发条件:
    · 任一币种偏离目标权重 > 15%
    · AI市场状态发生重大变化
    · 每5分钟强制检查一次


╔══════════════════════════════════════════════════════════════════════════════╗
║  M3: 蒙特卡洛回测引擎                                                        ║
╚══════════════════════════════════════════════════════════════════════════════╝

设计目标: 单次回测只是一个点估计, 无法反映策略在不同市场微观结构下的稳健性。
          通过蒙特卡洛模拟(Bootstrap重采样 + 参数扰动), 生成收益率的概率分布,
          计算VaR/CVaR, 给出更完整的风险评估。

━ 核心架构 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  增强: cryptoagents/backtest/engine.py (添加方法, 不破坏现有)

  方法: run_monte_carlo(n=100, noise_level=0.001)
  ┌─────────────────────────────────────────────┐
  │ 对同一段历史数据执行N次回测, 每次引入随机扰动: │
  │                                              │
  │ 随机因子:                                    │
  │   1. 滑点扰动:  slippage × N(1, noise)       │
  │   2. 信号跳过:   以p=5%概率跳过入场信号       │
  │                 (模拟实际交易中的执行延迟)     │
  │   3. 价格噪声:   收盘价 × N(1, noise/10)      │
  │                 (模拟成交价与回测价的偏差)     │
  │                                              │
  │ 每次运行记录:                                │
  │   {return, win_rate, max_drawdown,           │
  │    sharpe, trades, pf}                       │
  │                                              │
  │ 汇总指标:                                    │
  │   · Mean/Median return                       │
  │   · Std of returns (波动率)                   │
  │   · 95% CI: [P2.5, P97.5]                    │
  │   · Prob(profit): P(return > 0)              │
  │   · VaR_95: 95%置信度最大亏损                 │
  │   · CVaR_95: 超过VaR的平均亏损                │
  │   · Sharpe分布: [min, q25, median, q75, max] │
  │   · 收益率分布直方图数据                      │
  └─────────────────────────────────────────────┘

  API: POST /backtest/monte-carlo
  Request:  {strategy_id, symbol, start, end, capital, leverage, n=100, noise=0.001}
  Response: {summary, distribution, confidence_intervals, var_cvar, histogram_data}
